Key Analytical Findings

Six quantified insights across network analysis, forecasting, inventory, and risk domains

45.3%
Supply path length increase after 5-node attack
Removing just 5 high-betweenness suppliers (15.6% of nodes) disrupts 39% of all supply edges. Network fractures into disconnected components at Step 4 of the simulation.
84.4%
XGBoost accuracy improvement over ARIMA baseline
XGBoost achieves 2.05% MAPE vs ARIMA's 13.11% by incorporating 16 engineered features including demand lags, seasonality signals, part criticality, and supply context.
$4.37M
Annual inventory cost saving from EOQ optimisation
24% reduction across 196 SKUs. Critical finding: Class-C AOG parts are under-buffered in the current policy — cheap fasteners that ground aircraft are not being held in sufficient safety stock.
60.7%
Cost reduction from LP route optimisation
PuLP linear program reduces total transportation cost from $21.6M to $8.5M. 8 optimal routes replace 15 eligible routes — concentrating volume on lowest-cost suppliers within SLA constraints.
0.675
RF risk probability for GE Aviation — a blind spot
Rule-based scoring assigns GE Aviation MEDIUM risk (38.7). Random Forest surfaces 0.675 risk probability (near HIGH) due to high lead-time variability the weighted rule model under-penalises.
Parker
Hannifin — highest composite risk score (0.2351)
Single-source flag combined with 91.5% average OTD (below 92% benchmark) and 14+ day lead times. Dual-sourcing this one node would reduce AOG exposure for three downstream airlines simultaneously.

Project Timeline

10-week end-to-end delivery across 6 phases

Weeks 1–2
Phase 1: Research & Data Discovery
Audited 5 GitHub reference projects. Identified gap: no existing project combines aviation data + network graph + ML + live deployment. Selected MRO spare parts scope.
Weeks 2–3
Phase 2: Environment & ETL Pipeline
Built ingest.py + clean.py. Generated 500 SKU × 24 month synthetic MRO dataset. Configured GitHub Actions CI/CD (deploy.yml + lint.yml).
Weeks 3–5
Phase 3: Network Modelling
Built 32-node directed graph. Computed betweenness centrality, PageRank, composite risk scores. Simulated cascading failures. Exported Pyvis interactive network (726 KB).
Weeks 5–7
Phase 4: ML Models & Optimisation
ARIMA vs XGBoost demand forecasting (2.05% MAPE). EOQ inventory optimisation ($4.37M saving). PuLP LP route optimisation ($13.1M saving). Random Forest risk classifier (92% OOB).
Weeks 7–9
Phase 5: Dashboard & Deployment
5-page Streamlit dashboard. GitHub Pages landing site. PDF report via nbconvert. Streamlit Cloud deployment.
Weeks 9–10
Phase 6: Polish & Showcase
Unit tests. README badges + screenshots. LinkedIn post. Kaggle public notebook submission. Repo pinned.
